# 고객 포털 및 self-service 명세

## 1. 목적

본 문서는 고객 앱의 핵심 화면 책임, 문의 생성 흐름, 챗봇 self-service 흐름, 상태 확인 범위를 정의한다.

## 2. 고객 앱 기본 구조

- 고객 앱은 운영자 앱과 분리된 별도 앱 엔트리로 제공한다.
- 고객은 회원 로그인 후에만 문의와 챗봇 기능에 접근할 수 있다.

## 3. 고객 주요 기능

## 3.1 문의 탭

### 책임

- 본인 문의 목록 조회
- 문의 상세 및 메시지 확인
- 고객 메시지 발신
- 문의 상태 확인

### 규칙

- 고객은 본인에게 귀속된 문의만 조회할 수 있다.
- 고객은 문의 상태를 직접 변경할 수 없다.
- 고객은 고객 공개 대상 대화의 원문을 확인할 수 있다.
- 고객은 메시지당 최대 3장의 이미지를 첨부할 수 있다.
- 허용 형식은 `jpg`, `jpeg`, `png`이며, 이미지 1개당 최대 크기는 5MB다.

## 3.2 챗봇 탭

### 책임

- self-service 챗봇 세션 시작
- 챗봇 응답 확인
- FAQ/프리셋형 질문 사용
- 상담 전환 진입

### 규칙

- 챗봇 세션은 `chatbot_session`으로 별도 저장한다.
- 목록은 `문의` 탭과 `챗봇` 탭으로 분리해 표시한다.
- 고객은 본인 챗봇 세션 원문을 확인할 수 있다.
- 고객은 챗봇 대화 메시지에도 메시지당 최대 3장의 이미지를 첨부할 수 있다.
- 허용 형식은 `jpg`, `jpeg`, `png`이며, 이미지 1개당 최대 크기는 5MB다.

## 4. 상담 전환 흐름

- 고객은 챗봇 세션 중 언제든 상담 전환 버튼을 사용할 수 있다.
- 버튼 클릭 시 `inquiry`를 즉시 생성한다.
- 최근 챗봇 대화 또는 요약은 운영자가 확인할 수 있는 연결 맥락으로 남긴다.
- 유의미한 기록이 없고 고객 추가 메시지도 없으면 10분 후 자동 만료 대상으로 둘 수 있다.

## 5. 고객 상태 확인 범위

- 고객은 본인 문의의 현재 상태와 대화 내용을 확인할 수 있다.
- 기본 화면에는 상태, 대화 내용, 연결된 주문/상품의 요약 정보를 표시한다.
- 더 자세한 주문/상품 정보는 `[상세 보기]`를 통해 확장 조회할 수 있다.

## 6. 예외 처리

- 본인 소유가 아닌 문의/세션에 접근하려는 경우 `권한 없음` 처리
- 챗봇 응답 실패 시 self-service 실패 안내와 상담 전환 경로 제공
- 연결 데이터가 누락돼도 문의 목록과 대화 열람은 유지

## 7. 추가 정의 필요 사항

- 첨부 이미지 미리보기 방식
