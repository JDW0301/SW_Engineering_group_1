# AI 보조 및 자동화 기능 / 지식 자산 명세

## 1. 목적

본 문서는 요약, 자동 감지, self-service 챗봇, 상담 전환, FAQ/정책/프리셋 등 자동화 관련 기능과 지식 자산의 책임을 정의한다.

## 2. MVP 기능 범위

MVP에서 포함하는 자동화 범위는 아래 세 가지다.

1. `summary_assistant`: 문의 또는 챗봇 대화 요약
2. `abuse_detector`: 사전(JSON) 기반 부적절 표현 감지 및 표시 완화
3. `store_info_chatbot`: 스토어/상품 정보 기반 self-service 챗봇

다음 항목은 MVP 범위에서 제외한다.

- 범용 답변 추천 엔진
- 자유 생성형 상담 대체
- 광범위한 AI 자동응답 자동화

## 3. 지식 자산 분류

| 자산 | 목적 | 수정 주체 |
|---|---|---|
| `faq_article` | 반복 질문에 대한 표준 답변 | merchant_operator |
| `policy_document` | 배송/환불/교환 등 운영 정책 | merchant_operator |
| `response_preset` | 운영자가 빠르게 삽입하는 응답 문구 | merchant_operator |
| `chatbot_knowledge_file` | 챗봇 self-service 응답에 사용하는 스토어/상품 정보 파일 | merchant_operator |

MVP에서 챗봇 지식 파일은 아래 정책을 따른다.

- 형식: `.txt`만 허용
- 파일 크기: 1개당 최대 3MB
- 활성 방식: 여러 파일 동시 활성 허용
- 충돌 규칙: 동일 주제 충돌 시 최신 업로드 파일 우선
- 업로드 실패 시 기존 활성 파일 집합 유지
- 활성 파일 개수는 학교 프로젝트 기준으로 제한을 두지 않는다.
- 업로드 UI는 다중 파일 업로드를 지원한다.
- 운영자는 현재 활성 파일 목록을 확인할 수 있다.
- 다중 업로드 결과는 파일별 성공/실패를 개별 표시한다.

## 4. 문의/챗봇 요약 규칙

## 4.1 목적

- 운영자가 문의 또는 챗봇 대화 핵심을 빠르게 파악하도록 돕는다.

## 4.2 개입 위치

- 문의/챗봇 목록 카드 또는 세션 진입 전 맥락 파악 영역

## 4.3 입력

- 고객 메시지 원문
- 최근 대화 이력 일부

## 4.4 동작 방식

- 운영자가 요약 버튼을 눌렀을 때 로컬 LLM API를 호출한다.
- 생성된 요약 결과는 저장한다.
- 요약은 문의 또는 챗봇 세션당 최신 1개만 유지한다.
- 재생성 시 기존 요약은 최신 값으로 덮어쓴다.
- 재생성은 현재 전체 대화 기준으로 수행한다.

## 4.5 출력

- 핵심 이슈 요약
- 요청 사항
- 주문/상품/배송 관련 키워드

## 4.6 예외 처리

- 요약 실패 시 원문 메시지를 그대로 보여준다.
- 요약이 부정확할 수 있으므로 원문 접근은 항상 유지한다.

## 5. 자동 감지 규칙

## 5.1 목적

- 부적절 표현을 감지해 기본 표시를 완화하고 경고를 제공한다.

## 5.2 개입 위치

- 고객 공개 대상 대화 영역 내부

## 5.3 입력

- 고객 메시지 원문
- JSON 형태의 감지 사전

## 5.4 출력

- 감지 여부
- 감지 유형 목록
- 기본 표시용 마스킹/완화 결과
- 경고 메시지

## 5.5 규칙

- 감지는 LLM 추론이 아니라 사전 기반 탐지/치환을 사용한다.
- 기본 표시값은 `content_display`를 사용한다.
- 고객과 운영자 모두 고객 공개 대상 대화 영역의 원문을 확인할 수 있다.
- 감지 규칙과 사전 관리 주체는 운영자다.

## 5.6 예외 처리

- 감지 실패 시 분석 불가 상태를 표시한다.
- 원문 보기 기능은 감지 성공/실패와 무관하게 유지한다.

## 6. 스토어 정보 기반 챗봇 규칙

## 6.1 목적

- 고객이 사람 상담 전에 기본 정보를 self-service 방식으로 확인하도록 돕는다.

## 6.2 위치

- 고객 앱의 별도 self-service 진입점

## 6.3 입력

- 업로드된 스토어/상품 정보 파일
- FAQ 및 preset형 버튼 정보
- 고객 질문 텍스트

## 6.4 출력

- 제한된 범위의 안내 응답
- 상담 전환 진입점

## 6.5 규칙

- 챗봇 답변 범위는 업로드된 스토어/상품 정보와 FAQ 버튼 범위로 제한한다.
- 답변 생성 시 로컬 LLM API를 호출할 수 있다.
- 고객은 원하면 언제든 상담 전환 버튼으로 문의 흐름으로 이동할 수 있다.
- 범용 질의응답이나 자유 생성형 상담 대체는 MVP 범위로 보지 않는다.
- 여러 활성 지식 파일에 같은 주제의 충돌이 있으면 최신 업로드 파일을 우선한다.

## 7. 상담 전환 규칙

- 상담 전환 버튼 클릭 시 `inquiry`를 즉시 생성한다.
- 최근 챗봇 대화 또는 요약은 문의에 연결된 맥락으로 남긴다.
- 운영자는 연결된 `chatbot_session` 전체 원문을 참조 조회할 수 있어야 한다.
- 유의미한 기록이 없고 고객 추가 메시지도 없으면 10분 후 자동 만료할 수 있다.

## 8. 운영 책임

- 자동화 기능은 최종 결정 주체가 아니라 보조 기능이다.
- FAQ, 정책, 프리셋, 챗봇 지식 파일은 운영자가 갱신할 수 있어야 한다.
- 상담 응답의 최종 책임은 운영자에게 있다.

## 9. 로그 및 추적

- 요약 생성 이력
- 자동 감지 결과 이력
- 챗봇 세션 이력
- 상담 전환 이력
- 참조한 지식 자산 버전 정보

로그 보존 기간은 플랫폼 정책을 따른다.

## 10. 모듈 책임

- `knowledge_service`: FAQ, 정책, 프리셋, 챗봇 지식 파일 관리
- `summary_service`: 요약 생성
- `abuse_detection_service`: 감지 결과 생성
- `chatbot_service`: 챗봇 세션과 응답 생성
- `handoff_service`: 사람 상담 전환 이벤트 관리

## 11. 추가 정의 필요 사항

- 감지 사전의 형식과 배포 방식
