# 기능 명세

## 1. 기능 구조 개요

본 서비스의 기능은 크게 다음 영역으로 구분된다.

1. 계정 및 권한 관리
2. 스토어·상품·고객·주문 데이터 관리
3. 문의 및 상담 운영
4. 실시간 채팅
5. AI 상담 보조
6. 검색 및 운영 지표

아래 내용은 원문을 기반으로 기능 관점에서 재구성한 초안 명세이다. 세부 동작, 상태값, 정책 기준은 일부 문서화 편의를 위해 정리한 것이며, 원문상 확정되지 않은 항목은 별도로 표시했다.

## 2. 계정 및 권한 관리

### 2.1 로그인 및 인증

- 로그인 기능을 제공한다.
- 사업자 관리자 계정과 고객 계정을 구분한다.
- 인증 기능을 통해 사용자 유형별 접근을 통제한다.

### 2.2 접근 권한

- 사업자는 자신이 관리하는 스토어 정보에만 접근할 수 있어야 한다.
- 고객은 자신의 문의와 관련된 범위 내에서만 정보 접근이 가능해야 한다.
- 세부 권한 수준(예: 관리자 내부 세분화 여부)은 **추가 구체화 필요**하다.

## 3. 스토어·상품·고객·주문 관리

### 3.0 통합 상담 조회 화면

- 사업자는 상담 시 스토어, 상품, 주문, 고객, 문의 관련 정보를 분리된 화면이 아닌 연결된 맥락으로 확인할 수 있어야 한다.
- 이 통합 조회 구조는 원문에서 반복적으로 드러나는 핵심 요구사항이며, 실제 화면 구성 방식은 **추가 구체화 필요**하다.

### 3.1 스토어 정보 관리

- 스토어 정보를 등록, 조회, 수정할 수 있는 기능이 필요하다.
- 스토어 검색 및 선택 기능을 제공해야 한다.

### 3.2 상품 정보 관리

- 상품 정보 저장을 위한 DB 구조가 필요하다.
- 상품 등록 및 조회 기능이 필요하다.
- 상품 데이터의 수정·비활성화 범위는 **확인 필요**하다.

### 3.3 고객 정보 관리

- 고객 정보 저장을 위한 DB 구조가 필요하다.
- 고객 정보 조회 기능이 필요하다.
- 고객 단위로 주문 이력과 문의 이력을 연결 조회할 수 있어야 한다.

### 3.4 주문 정보 관리

- 주문 정보의 등록 및 조회 기능이 필요하다.
- 주문 상태 변경 기능은 원문에 언급되어 있으나, 실제 운영 범위는 **추가 구체화 필요**하다.
- 주문번호 기준 조회가 가능해야 한다.

## 4. 문의 및 상담 운영 기능

### 4.1 문의 접수

- 소비자가 문의를 등록할 수 있어야 한다.
- 문의 유형을 지정하거나 분류할 수 있어야 한다.
- 문의 유형 예시는 배송, 환불, 교환, 상품 문의 등이다.

### 4.2 문의 상태 관리

- 원문에는 다음 상태가 예시 수준으로 제시되어 있다.
  - 접수
  - 대기
  - 진행 중
  - 종료
  - 보류
- 상태 전환 권한, 필수 조건, 자동 전환 여부는 **추가 구체화 필요**하다.

### 4.3 문의 목록 및 정렬

- 문의 게시판 형식의 목록을 제공한다.
- 최근 문의순, 미처리 문의 우선 정렬 등 기본 정렬 기능이 필요하다.
- 상태별 필터링을 지원한다.

### 4.4 상담 이력 및 메모

- 채팅 이력 또는 문의 응답 이력을 저장한다.
- 관리자가 내부 확인용 메모를 남길 수 있어야 한다.
- 고객에게 노출되지 않는 내부 메모와 공개 응답은 구분되어야 한다.

### 4.5 응답 프리셋

- 상담 상황에 따라 사용할 수 있는 커스텀 프리셋 응답 기능을 제공한다.
- 자주 사용하는 응답 문구를 등록·수정·선택할 수 있어야 한다.
- 프리셋의 카테고리 구조는 **확인 필요**하다.

## 5. 실시간 채팅 기능

### 5.1 채팅 기본 기능

- 고객과 관리자 간 실시간 채팅 기능을 제공한다.
- 채팅방 생성 기능을 제공한다.
- 채팅 이력을 저장한다.

### 5.2 파일 전송

- 이미지 전송 기능을 제공한다.
- 영상 전송 기능을 제공한다.
- 파일 저장 방식은 별도 정책이 필요하다.
- 파일 크기 제한 기준은 **확인 필요**하다.
- 첨부 파일 미리보기 제공 여부와 범위는 **추가 구체화 필요**하다.

### 5.3 상담 진행 상태 표시

- 읽음 여부 및 마지막 응답 시간 표시 기능은 원문에 언급되어 있으므로 포함 가능성이 높다.
- 실제 표시 위치와 사용자별 노출 방식은 **추가 구체화 필요**하다.

## 6. AI 상담 보조 기능

### 6.1 사업자 맞춤형 AI 챗봇

- 사업자 맞춤형으로 설정 가능한 AI 챗봇 기능을 제공한다.
- 반복 문의를 우선 자동 응대할 수 있어야 한다.
- FAQ, 정책, 답변 기준 문서를 참조할 수 있는 구조가 필요하다.

### 6.2 답변 지식 관리

- 자주 묻는 질문을 등록하고 수정할 수 있어야 한다.
- 배송, 환불, 교환 정책을 관리할 수 있어야 한다.
- 챗봇 답변 기준 문서를 관리할 수 있어야 한다.
- 금지 답변 및 예외 답변 규칙은 원문에 언급되어 있으나, 실제 정책 단위와 관리 방식은 **추가 구체화 필요**하다.

### 6.3 메시지 요약

- 고객 메시지의 핵심 내용을 요약해야 한다.
- 요약 결과는 관리자가 빠르게 내용을 파악할 수 있는 형태여야 한다.
- 원문에서 언급된 “중립적 표현 변환”은 기능 의도는 분명하지만 출력 방식은 **추가 구체화 필요**하다.

### 6.4 악성 표현 감지 및 필터링

- 욕설, 모욕, 인신공격, 혐오 표현을 감지해야 한다.
- 감지된 표현에 대한 필터링 또는 완화 표시 기능이 필요하다.
- 감지 결과를 차단할지, 요약으로 대체할지, 경고만 표시할지는 **확인 필요**하다.

### 6.5 관리자 응대 전환

- AI가 응답하지 못하거나 부적절한 경우 관리자 응대로 전환해야 한다.
- 전환 조건과 기준은 명확히 정의할 필요가 있다.

## 7. 검색 및 운영 지표

### 7.1 검색 기능

- 주문번호 기준 검색
- 고객명 기준 검색
- 상품명 기준 검색
- 문의유형 기준 검색
- 기간별 검색

### 7.2 운영 지표

- 일별 문의 수
- 주별 문의 수
- 월별 문의 수
- 평균 응답 시간
- 해결 완료율

위 지표는 원문에 언급된 운영 지표 초안이며, 계산 기준과 집계 주기는 **추가 정의 필요**하다.

## 8. 기능 우선순위 제안

아래 우선순위는 원문을 토대로 재구성한 **편집자 제안**이며, 실제 일정 계획과 동일하지 않다.

### 1순위

- 로그인 및 계정 구분
- 스토어/고객/주문/문의 기본 관리
- 문의 목록 및 상태 관리
- 상담 기록 저장

### 2순위

- 실시간 채팅
- 파일 첨부
- 프리셋 응답
- 검색 및 필터링

### 3순위

- 사업자 맞춤형 AI 챗봇
- 고객 메시지 요약
- 악성 표현 감지 및 필터링
- 운영 지표 및 리포트

우선순위는 원문의 문제 정의와 구현 부담을 함께 고려한 제안이며, 실제 일정 계획과 동일한 것은 아니다.
