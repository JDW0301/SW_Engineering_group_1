# Immediate MVP Notes

## 문서 목적

이 문서는 **MVP 범위를 논의한 시점 이후** 기준으로, 현재까지 확정된 결정과 아직 세부 파라미터만 남은 항목을 빠르게 정리하기 위한 실행 메모이다.

본 문서는 상세 명세의 대체 문서가 아니라, 현재 의사결정과 후속 작업을 빠르게 정렬하기 위한 실행 메모로 사용한다.

---

## 1. 현재까지 확정된 방향

### 1.1 사용자 범위

- MVP 1차 출시 대상에는 **운영자와 고객**을 모두 포함한다.
- 고객은 **회원 로그인 기반**으로만 문의/챗봇 기능을 사용한다.

### 1.2 플랫폼 방향

- UI는 **React 기반 웹**으로 구현한다.
- 고객 앱과 운영자 앱은 **앱 엔트리 2개로 분리**한다.
- 다만 백엔드, 도메인 모델, 저장소는 하나의 Modular Monolith 안에서 공유한다.

### 1.3 자동화/AI 범위

- MVP 자동화 범위는 아래 세 가지로 고정한다.
  - 문의/챗봇 대화 요약
  - 사전 기반 악성 표현 감지
  - 스토어/상품 정보 기반 self-service 챗봇
- 아래 항목은 후순위로 둔다.
  - 권한 세분화
  - AI 추천 답변
  - 자유 생성형 상담 대체

### 1.4 고객 기능 범위

- 고객은 아래 기능을 사용할 수 있어야 한다.
  - 문의 등록 및 메시지 발신
  - 챗봇 self-service 사용
  - 상태 확인
  - 상담 전환

### 1.5 운영자 핵심 기능 범위

- 문의 목록/상세
- 통합 조회
- 상태 관리
- 프리셋 응답
- 검색/필터
- 운영 지표
- 외부 데이터 수동 재동기화

## 2. 현재 기준 MVP 해석

### 2.1 핵심 제품 형태

- 자사몰 고객센터 운영을 위한 **운영자 앱 + 고객 앱 구조의 통합 웹 서비스**
- 운영자는 문의를 관리하고 응대한다.
- 고객은 챗봇 self-service를 먼저 사용하거나 필요 시 문의로 전환한다.

### 2.2 데이터 구조 방향

- 외부 시스템 역할을 하는 **가상의 외부 DB**가 존재한다.
- 프로젝트 DB는 외부 DB의 모든 데이터를 복제하는 저장소가 아니라, **필요한 정보만 반영하는 운영용 DB**로 사용한다.
- 데이터 반영은 기본적으로 **웹훅 기반 이벤트 반영**을 전제로 하고, 운영자는 고객/주문/상품 단위 수동 재동기화를 수행할 수 있다.

### 2.3 운영 경험 방향

- 운영자는 문의를 볼 때 고객, 주문, 상품, 문의 이력을 한 맥락으로 확인할 수 있어야 한다.
- 챗봇 self-service에서 사람 상담으로 전환된 경우, 운영자는 연결된 챗봇 원문 맥락을 확인할 수 있어야 한다.
- 고객 공개 대상 대화는 기본적으로 완화된 표시값을 사용하되, 고객과 운영자 모두 원문을 확인할 수 있다.

## 3. 현재 기준 확정된 정책 요약

1. 서비스 구조: 고객 앱 / 운영자 앱 2엔트리 분리
2. 고객 식별: 회원만 허용
3. 원본 관계: 외부 원본 + 내부 최소 snapshot
4. 상태 모델: `OPEN`, `WAITING`, `IN_PROGRESS`, `ON_HOLD`, `RESOLVED`
5. 운영 보조 신호: 최소 유지 (`assigned_operator_id`, `last_message_at`, `last_response_at`)
6. 메시지 저장: `content_raw` / `content_display` 분리
7. 챗봇 구조: `chatbot_session` 별도 엔티티, `문의`/`챗봇` 2탭 구성
8. 상담 전환: 즉시 `inquiry` 생성 + 챗봇 맥락 연결 + 무의미 세션 자동 만료 가능
9. 권한 구조: MVP 단일 운영자 역할
10. 운영 지표: 문의 수, 평균 응답 시간, 해결 완료율
11. 예외 처리: 사용자 노출 중심 단순 오류 체계
12. 요약 정책: 최신 1개 유지, 덮어쓰기, 전체 대화 기준 재생성
13. 챗봇 지식 파일: `.txt`, 파일당 3MB, 다중 활성 허용, 충돌 시 최신 업로드 우선, 업로드 실패 시 기존 활성 집합 유지
14. 고객 상태 화면: 기본은 상태/대화/요약 정보, 상세는 `[상세 보기]`로 확장
15. 인증 방식: 세션 기반 인증, 브라우저 종료 시 만료, 비활성 타임아웃 없음
16. 지식 파일 업로드 UI: 다중 업로드, 활성 파일 목록 표시, 파일별 성공/실패 표시
17. 첨부 이미지: 썸네일 표시, 클릭 확대, 실패 시 다운로드만 제공
18. 운영자 검색: 고객명, 주문번호, 상태, 문의 유형, 기간, 상품명
19. 지표 UI: 평균 응답 시간은 시간/분 자동 변환, 집계 0건 시 평균/해결율은 `-` 표시
20. 웹훅 세부: 진위 검증 생략, 일시 실패 시 최대 3회 재시도
