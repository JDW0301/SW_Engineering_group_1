# 프로젝트 문서 안내

이 폴더는 소규모 사업자의 자사몰 고객센터 운영을 지원하는 플랫폼 프로젝트 문서를 체계적으로 정리한 공간이다. 원본 PDF에 포함된 문제 정의, 해결 방향, 사용자 분석, 기능 요구사항, 구현 메모를 문서 목적에 맞게 재구성하였다.

## 문서 구성

- [01. 프로젝트 개요](./01-project-overview.md)
  - 프로젝트 배경, 문제 정의, 해결 방향, 기대 효과를 정리한다.
- [02. 사용자 및 가치 분석](./02-user-needs-and-value.md)
  - 주요 사용자군, 사용자 니즈, 제공 가치, 핵심 사용 시나리오를 정리한다.
- [03. 범위 및 요구사항](./03-scope-and-requirements.md)
  - 프로젝트 목표, 범위, 기능/비기능 요구사항, 제외 범위를 정리한다.
- [04. 기능 명세](./04-functional-specification.md)
  - 실제 서비스에 필요한 기능을 도메인별로 세분화하여 정리한다.
- [05. 운영·보안·기술 메모](./05-operations-security-and-technical-notes.md)
  - 운영 정책, 보안 요구사항, 데이터 연동, 기술 스택 및 구현 메모를 정리한다.
- [06. 로드맵 및 확인 필요 사항](./06-roadmap-and-open-issues.md)
  - 단계별 추진 방향, 후속 과제, 현재 기준으로 추가 확인이 필요한 항목을 정리한다.

## 정리 원칙

- 원문에 흩어져 있던 내용을 문서 목적에 맞게 재배치했다.
- 러프한 표현은 문서체로 다듬고, 가능한 범위에서 의미를 명확히 했다.
- 원문에 근거가 부족한 내용은 단정하지 않고 **확인 필요** 또는 **추가 구체화 필요**로 표기했다.
- 아직 수치 기준, 일정, 정책 세부안이 없는 항목은 후속 결정 대상으로 분리했다.

## 문서 활용 권장 순서

1. `01-project-overview.md`로 프로젝트 배경과 목적을 이해한다.
2. `02-user-needs-and-value.md`로 대상 사용자와 핵심 가치를 정리한다.
3. `03-scope-and-requirements.md`와 `04-functional-specification.md`를 기준으로 구현 범위를 확정한다.
4. `05-operations-security-and-technical-notes.md`로 운영·보안·기술 의사결정을 보완한다.
5. `06-roadmap-and-open-issues.md`를 바탕으로 다음 단계 작업을 계획한다.
