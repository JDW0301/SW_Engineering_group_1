# 제품 범위 및 명세 기준

## 1. 문서 목적

본 문서는 현재 기준 제품 범위와 구현 명세 작성 기준을 정의한다. 다른 모든 상세 명세 문서는 본 문서의 범위 선언을 벗어나지 않아야 한다.

## 2. 제품 정의

본 서비스는 **소규모 온라인 사업자의 자사몰 고객센터 운영을 지원하는 통합 CS 플랫폼**이다.

핵심 가치는 다음과 같다.

- 고객, 주문, 상품, 문의 정보를 연결한 통합 상담 맥락 제공
- 고객이 스스로 해결을 시도할 수 있는 챗봇 기반 self-service 진입점 제공
- 메시지 요약과 자동 감지 기능으로 상담자 부담 완화
- 소수 인원으로도 운영 가능한 고객센터 도구 제공

## 3. 현재 기준 제품 구조

- 제품은 **하나의 Modular Monolith** 안에서 운영한다.
- 사용자 표면은 아래 두 개의 앱 엔트리로 분리한다.
  - `customer-app`: 고객용 앱
  - `operator-app`: 운영자용 앱
- 두 앱은 같은 백엔드, 같은 도메인 모델, 같은 저장소를 공유하지만 라우팅과 사용자 경험은 분리한다.

## 4. 현재 기준 범위 정의

## 4.1 포함 범위

### 필수 포함

- 운영자 기준 상담 작업공간
- 회원 고객 전용 고객 앱
- 문의 접수 및 상태 관리
- 고객/주문/상품/문의 통합 조회
- 상담 기록 및 내부 메모
- 기본 검색 및 필터링
- 운영 지표
- 역할 기반 접근 제한
- FAQ/정책/프리셋 기반 운영 지원
- 문의 맥락 파악용 메시지 요약
- 사전 기반 악성 표현 감지 및 표시 완화
- 스토어/상품 정보 기반 self-service 챗봇
- 챗봇에서 문의로의 상담 전환 구조
- 외부 시스템 웹훅 기반 반영과 운영자 수동 재동기화

### 포함 가능성이 높으나 추가 상세화 필요

- 실시간 채팅 전송 보장 수준
- 첨부 파일 미리보기 범위
- 챗봇 지식 파일 업로드 UI 세부 방식

## 4.2 제외 또는 보류 범위

다음 항목은 현재 기준으로 MVP 핵심 범위 밖으로 본다.

- 비회원 문의 접수
- 관리자/상담자 역할 세분화
- 범용 답변 추천 엔진
- 자유 생성형 상담 대체 AI
- 대형 종합 커머스 플랫폼 전체 기능 대체
- 마케팅 자동화 기능
- 재고·정산·광고 통합 운영 도구
- 고급 분석 BI 체계

## 5. 사용자 범위

## 5.1 사용자 유형

- `merchant_operator`: 스토어 고객센터 운영 주체
- `customer`: 로그인한 회원 고객. 본인 문의와 챗봇 세션에만 접근 가능

관리자/상담자 역할 세분화는 MVP 후속 확장 항목으로 둔다.

## 5.2 사용자별 핵심 목표

- 운영자: 문의를 빠르게 분류하고 정확히 응대하며 운영 부담을 줄인다.
- 고객: 챗봇 또는 문의를 통해 주문/상품 관련 문제를 빠르게 해결받는다.

## 6. 명세 작성 기준

## 6.1 명명 규칙

- 문서 내 엔티티 명은 영문 snake_case 기준 식별자와 한국어 표시명을 함께 사용한다.
  - 예: `inquiry_status(문의 상태)`
- 상태값은 영문 상수형 표기와 한국어 설명을 함께 기록한다.
  - 예: `OPEN(접수됨)`
- 화면/모듈 명은 영문 kebab-case 파일명 기준과 한국어 화면명을 매핑한다.

## 6.2 확정 수준 표기

- `확정`: 현재 문서 기준으로 구현 가능 수준까지 고정된 항목
- `추가 정의 필요`: 방향은 있으나 세부 규칙이 부족한 항목
- `정책 결정 필요`: 사업/운영 판단이 선행되어야 하는 항목
- `확인 필요`: 원문 또는 입력 자료상 근거가 불충분한 항목

## 7. MVP 완료 기준

현재 문서 기준 MVP는 아래 조건을 만족해야 한다.

1. 운영자가 문의 목록과 상세를 통해 고객/주문/상품/문의 맥락을 확인할 수 있다.
2. 문의 상태를 관리하고 상담 기록 및 내부 메모를 남길 수 있다.
3. 기본 검색과 필터링이 가능하고 문의 수, 평균 응답 시간, 해결 완료율을 확인할 수 있다.
4. 고객은 로그인 후 챗봇 또는 문의를 통해 문제를 제기하고 상태를 확인할 수 있다.
5. 스토어/상품 정보 기반 챗봇이 self-service 진입점으로 동작하고, 고객이 원하면 즉시 상담 전환이 가능하다.
6. 운영자는 요약과 자동 감지 결과를 참고할 수 있다.
7. 외부 고객/주문/상품 데이터는 웹훅 기반으로 반영되고, 운영자는 개체 단위 수동 재동기화를 수행할 수 있다.

## 8. 후속 상세화가 필요한 범위

- 모바일 앱 포함 여부
- 외부 시스템 연동 대상 구체화
- 실시간 채팅 전송 보장 수준
- 챗봇 지식 파일 갱신 UI 및 운영 세부
