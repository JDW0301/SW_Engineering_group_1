# 간소화된 명세 문서 세트

이 폴더는 프로젝트를 대학교 수준에서 다시 구현하기 위해 새롭게 정리한 명세 문서 세트다.

이 문서들의 목표는 다음과 같다.

- 원래 프로젝트의 주제와 방향은 유지하고,
- 주요 기능 묶음은 그대로 살리며,
- 실제로 작동하는 프로젝트를 만드는 데 꼭 필요하지 않은 엔터프라이즈급 운영 디테일은 덜어낸다.

## 제품 요약

이 제품은 소규모 온라인 판매자를 위한 웹 기반 고객지원 시스템이다.

- `customer-app`: 고객이 문의를 등록하고, 챗봇을 사용하며, 메시지를 보내고, 진행 상태를 확인한다.
- `operator-app`: 운영자가 문의를 관리하고, 고객/주문/상품 맥락을 확인하며, 응답하고, 지식/안전 보조 기능을 활용한다.

## 역할 용어 정리

이 명세에서는 `운영자`, `관리자`, `상담자`를 병용할 수 있다.

- `운영자`는 주로 `operator-app`에서 일하는 사용자 흐름과 화면 문맥을 가리킨다.
- `관리자`는 주로 권한, 계정, 응대 책임 문맥을 가리킨다.
- `상담자`는 고객 응대 행위를 설명할 때 쓰는 표현이지만, 별도 역할을 뜻하지 않는다.
- 세 표현은 서로 다른 사람이 아니라, **관리자 계정으로 로그인해 운영자 앱에서 상담을 처리하는 같은 사람**을 뜻한다.

## 이 명세가 의도적으로 단순화하는 것

이 명세는 아래 항목을 완전하게 정의하려 하지 않는다.

- 상세한 복구 워크플로,
- 복잡한 권한 매트릭스,
- 이벤트 로그 운영,
- 재시도/백오프 전략,
- 깊은 검증 분기,
- 감사/보관 거버넌스,
- 저수준 저장소 및 인프라 동작.

대신 아래에 집중한다.

- 눈에 보이는 화면,
- 핵심 사용자 흐름,
- 필수 데이터 관계,
- 실제로 동작하는 구현에 필요한 최소 행동 정의.
