# 대화와 콘텐츠

## 목적

이 기능은 inquiry와 chatbot 대화가 어떻게 표시되고 갱신되는지를 정의한다.

또한 첨부파일을 기능 수준에서 어떻게 다루는지도 정의한다.

## 대화 유형

### 문의 대화

문의 안에서 고객과 운영자가 주고받는 메시지다.

### 챗봇 대화

챗봇 세션 안에서 고객과 챗봇이 주고받는 메시지다.

## 공통 표시 원칙

대화 UI는 아래를 보여줘야 한다.

- 메시지 타임라인,
- 발신자 구분,
- 표시용 콘텐츠,
- 선택적 raw-content 접근,
- 첨부 썸네일 또는 미리보기,
- 현재 사용자 기준 메시지 입력 영역.

## 첨부파일

이 프로젝트는 이미지 첨부를 지원한다.

최소 기대 동작:

- 이미지 업로드,
- 메시지 아래 첨부 표시,
- 이미지 미리보기,
- 메시지와의 연결 유지.

구현에서는 `message_attachment` 같은 최소 단위의 별도 구조를 두는 것을 기본으로 본다.

정확한 파일 검증 규칙이나 저장소 내부 구현은 이 명세의 중심에 두지 않는다.

## 메시지 콘텐츠 모델

메시지는 두 가지 텍스트 뷰를 사용한다.

- `content_raw`: 원본 내용
- `content_display`: 고객에게 보여주는 내용

이 구조는 원문을 잃지 않으면서 summary/safety 동작을 지원한다.

## 주요 흐름

### 문의 메시지 흐름

1. 고객 또는 운영자가 메시지를 작성한다.
2. 메시지가 저장된다.
3. 문의 타임라인이 갱신된다.
4. 필요 시 문의 시간 필드가 갱신된다.

### 챗봇 메시지 흐름

1. 고객이 질문을 입력한다.
2. chat message가 저장된다.
3. 챗봇 응답이 생성된다.
4. 챗봇 타임라인이 갱신된다.

## 단순화 메모

이 명세는 아래를 완전하게 정의하지 않는다.

- websocket 전송 보장,
- 고급 미리보기 fallback 분기,
- delivery-state 매트릭스,
- exhaustive한 검증 규칙,
- 저장소 구현 디테일.
