# 문서 버전 및 참조 정책

## 1. 목적

본 문서는 상세 명세 문서를 읽고, 수정하고, 새 버전을 작성할 때 따라야 하는 공통 버전 참조 원칙을 정의한다.

## 2. 기본 원칙

- 각 주제/영역별 **최신 active 문서**를 기본 기준 문서로 사용한다.
- 과거 버전 문서는 항상 보존하되, 기본 작업 기준은 과거 문서가 아니라 최신 active 문서이다.
- 문서 수정 또는 신규 버전 작성 시에는 과거 문서를 처음부터 순차적으로 훑지 않는다.
- 먼저 현재 active 문서를 읽고, 필요성이 생길 때만 관련 과거 문서를 선택적으로 참조한다.

## 3. 과거 버전 문서를 참조하는 예외 조건

과거 버전 문서는 아래 경우에만 선택적으로 참조한다.

1. 사용자가 특정 버전 또는 이전 문서를 직접 보라고 지시한 경우
2. 현재 문서만으로 결정 근거, 변경 이유, 정책 변화 이력을 확인할 수 없는 경우
3. 현재 내용이 이전 버전과 충돌하거나, 누락/변경 여부를 비교 검토해야 할 필요가 있는 경우
4. 회귀 여부, 변경 영향, 과거 결정 배경을 확인해야 하는 경우

## 4. 과거 버전 문서를 읽었을 때의 기록 규칙

과거 버전을 읽은 경우에는 작업 결과에 아래 내용을 간단히 남긴다.

- 왜 읽었는지
- 어떤 문서를 참고했는지
- 현재 작업에 어떤 영향을 주는지

## 5. 버전 작성 원칙

- 새 버전 문서는 이전 파일을 삭제하거나 덮어쓰지 않고 보존한 채 생성한다.
- 단, 현재 기준 문서는 항상 `active/`에서 가장 쉽게 찾을 수 있어야 한다.
- 과거 버전은 `archive/` 또는 그에 준하는 보관 영역에서 추적한다.
- 읽는 사람은 먼저 `active/`를 보고, 비교가 필요할 때만 `archive/`를 확인할 수 있어야 한다.

## 6. 현재 문서 우선 읽기 순서

1. 관련 주제의 `active/` 문서 확인
2. 현재 문서만으로 충분한지 판단
3. 부족하거나 충돌이 있는 경우에만 관련 과거 문서 확인
4. 필요 시 변경 이력과 영향 범위 정리

## 7. 적용 예시

### 예시 1. 일반 수정 작업

- `active/modules/ai-assistant-and-knowledge.md`를 먼저 읽는다.
- 현재 규칙만으로 수정 가능하면 과거 버전은 읽지 않는다.

### 예시 2. 정책 충돌 확인 작업

- `active/platform/integration-security-and-constraints.md`를 먼저 읽는다.
- 현재 보관 정책 근거가 불충분하면 `archive/` 또는 이전 기준 문서를 추가 확인한다.
- 이 경우, 왜 과거 문서를 읽었는지와 현재 정책에 미치는 영향을 함께 기록한다.

## 8. 결론

이 문서 세트의 기본 운영 원칙은 **최신 문서 우선, 과거 문서는 예외적으로만 선택 참조**이다.
