# 연동·보안·예외 처리 및 공통 제약

## 1. 목적

본 문서는 외부 연동, 보안, 개인정보, 보관 정책, 공통 예외 처리 규칙을 한 곳에서 정의한다.

## 2. 외부 연동 및 동기화

## 2.1 연동 대상

- 외부 스토어/주문 시스템을 연동 대상으로 본다.
- 구체 대상 시스템은 **추가 정의 필요**하다.

## 2.2 동기화 방식

- 기본 반영 방식은 `webhook event`다.
- 운영자는 `customer`, `order`, `product` 개체 단위로 수동 재동기화를 수행할 수 있다.
- 전체 스토어 단위 재동기화는 MVP 범위에서 제외한다.

## 2.3 동기화 규칙

- 동일 외부 이벤트의 중복 수신 가능성을 고려해야 한다.
- 재처리 안전성(idempotency) 확보가 필요하다.
- 수동 재동기화도 동일 개체에 대해 반복 실행 가능해야 한다.

## 2.4 충돌 처리

- 외부 snapshot 데이터의 최종 기준은 외부 원본이다.
- 내부 문의/메시지/자동화 결과 데이터는 본 시스템 원본이다.
- 외부 snapshot과 내부 운영 데이터는 직접 같은 엔티티를 공유하지 않도록 ACL을 둔다.

## 3. 개인정보 및 보안

## 3.1 데이터 분류 원칙

- 공개 가능 정보
- 고객 공개 대상 대화 정보
- 운영자 내부 정보
- 민감정보

민감정보 세부 필드 목록은 **추가 정의 필요**하다.

## 3.2 마스킹 규칙

- 고객 연락처와 주소류는 기본 마스킹 대상이다.
- 고객 공개 대상 대화는 기본적으로 `content_display`를 사용한다.
- 고객과 운영자는 고객 공개 대상 대화의 원문을 확인할 수 있다.
- 내부 메모와 내부 처리 데이터는 원문 공개 대상에서 제외한다.

## 3.3 접근 로그

- 고객정보 조회
- 문의 상세 조회
- 수동 재동기화 실행
- 정책/FAQ/지식 파일 수정
- 원문 보기 동작

위 행위는 감사 로그 후보이다.

## 3.4 보관/삭제 정책

- 탈퇴 후 데이터 보관 기간은 **추가 정의 필요**하다.
- 첨부 파일 보관 기간은 문의 데이터 정책과 정합성을 가져야 한다.
- 삭제와 익명화 중 어떤 방식을 택할지는 **추가 정의 필요**하다.

## 4. 공통 입력 제약

## 4.1 검색 입력

- 날짜 범위는 시작일이 종료일보다 늦을 수 없다.
- 주문번호는 허용 형식 검증 규칙이 필요하다.

## 4.2 메시지 입력

- 빈 문자열 메시지는 저장하지 않는다.
- 최대 길이 제한은 **추가 정의 필요**하다.

## 4.3 파일 입력

- 허용 형식이 아닌 파일은 차단한다.
- 크기 제한 초과 파일은 차단한다.

## 5. 공통 예외 처리 원칙

MVP의 예외 처리 원칙은 사용자 노출 중심의 단순 오류 체계를 따른다.

## 5.1 사용자 입력 오류

- 잘못된 형식 입력은 즉시 검증 오류로 반환한다.
- 어떤 항목이 잘못되었는지 사용자에게 명시한다.

## 5.2 권한 오류

- 인증 실패와 권한 실패를 구분한다.
- 리소스 존재 여부 노출 최소화 원칙을 따른다.

## 5.3 외부 연동 오류

- 웹훅 지연, 일시 오류, 수동 재동기화 실패를 구분해 안내할 수 있어야 한다.
- 재시도 가능 여부를 로그에 남긴다.

## 5.4 자동화 처리 오류

- 요약 실패: 원문 유지
- 자동 감지 실패: 분석 불가 상태 표시
- 챗봇 응답 실패: self-service 응답 불가 안내 또는 상담 전환 유도

## 5.5 누락 데이터 오류

- 주문/상품 매핑 실패 시에도 문의는 열람 가능해야 한다.
- 누락 데이터는 안내 메시지로 보완한다.

## 5.6 자동 만료

- 챗봇 세션 또는 챗봇 전환으로 즉시 생성된 문의는 유의미한 기록이 없고 고객 추가 메시지도 없을 때만 10분 후 자동 만료 대상으로 둘 수 있다.
- 일반 문의에는 자동 만료 규칙을 적용하지 않는다.

## 6. 모듈 책임 경계

- `integration_service`: 외부 데이터 수집, 재처리, 이벤트 추적
- `security_policy_service`: 마스킹, 원문 공개 기준, 접근 통제 기준 제공
- `retention_policy_service`: 보관/삭제 기준 제공
- `validation_service`: 공통 입력 검증
- `error_mapping_service`: 오류 유형을 사용자 노출 방식으로 변환

## 7. 구현 전 추가 정의 필요 사항

- 세션 만료 정책
- 재인증 정책
- 파일 저장소 선택
- AI/자동화 처리 결과 저장 기간
- 법적 보관 정책 반영 기준
