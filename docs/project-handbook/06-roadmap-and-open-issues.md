# 로드맵 및 확인 필요 사항

## 1. 단계별 추진 방향 제안

원문에는 구체적인 일정이나 마일스톤이 포함되어 있지 않으므로, 아래 내용은 현재 요구사항을 바탕으로 정리한 **편집자 제안 성격의 단계별 추진 방향 초안**이다. 즉, 원문의 직접 전사가 아니라 후속 계획 수립을 돕기 위한 정리 문서이다.

## 1단계: 기본 CS 운영 기반 구축

우선 구현이 필요한 범위는 서비스의 기본 운영이 가능한 수준의 핵심 기능이다.

- 로그인 및 사용자 구분
- 스토어/상품/고객/주문 데이터 구조 설계
- 문의 접수 및 상태 관리
- 고객별 주문 이력·문의 이력 조회
- 문의 목록, 검색, 필터링
- 상담 이력 저장 및 내부 메모

### 1단계 목표

- 자사몰 고객센터 운영의 기본 흐름을 시스템에서 처리할 수 있는 상태를 만든다.

## 2단계: 상담 생산성 향상 기능 추가

- 실시간 채팅 기능
- 이미지/영상 첨부
- 읽음 여부 및 마지막 응답 시간 표시
- 커스텀 프리셋 응답
- FAQ 및 정책 문서 관리

### 2단계 목표

- 상담자가 더 빠르고 일관되게 대응할 수 있는 운영 도구를 제공한다.

## 3단계: AI 보조 기능 도입

- 사업자 맞춤형 AI 챗봇
- 고객 메시지 요약
- 악성 표현 감지 및 필터링
- AI 응답 실패 시 상담자 전환

### 3단계 목표

- 반복 문의 자동화와 상담자 부담 완화를 통해 운영 효율을 높인다.

## 4단계: 운영 고도화

- 문의 통계 및 지표 제공
- 운영 정책 정교화
- 외부 서비스 연동(API, 웹훅) 고도화
- 보안 및 개인정보 정책 강화

### 4단계 목표

- 서비스 운영 안정성과 관리 효율을 높이고 확장 기반을 마련한다.

## 2. 우선 결정이 필요한 사항

다음 항목은 구현 전에 우선 확정하는 것이 좋다.

### 제품 범위

- MVP에서 반드시 포함할 기능 범위
- 웹만 우선 제공할지, 앱까지 포함할지 여부
- 사업자 전용 도구인지, 소비자 화면까지 함께 개발할지 여부

### 데이터 및 연동

- 외부 스토어/주문 시스템 연동 대상
- 동기화 주기와 실패 처리 정책
- 첨부 파일 저장 전략

### AI 기능 정책

- 챗봇이 참조할 지식 소스 범위
- 금지 답변 및 예외 답변의 판단 기준
- AI 응답 실패 판정 기준
- 악성 표현 감지 결과의 노출 방식

### 보안 및 운영 정책

- 개인정보 보관 기간
- 탈퇴 후 데이터 처리 정책
- 역할별 접근 권한 세분화
- 로그 및 이력 관리 정책

## 3. 확인 필요 항목 목록

## 3.1 원문 해석상 불명확한 항목

- `ReAct`가 실제 기술명인지, `React` 오기인지 확인 필요
- “시간이 괜찮으면 웹/앱 서비스 진행도 고려”의 정확한 의미 확인 필요
- “고객 응대 시 감정 소모 문제 방지를 위한 서비스”의 구체적 UI/정책 정의 필요
- “중립적 표현 변환” 결과를 어디에서 어떻게 사용할지 확인 필요

## 3.2 요구사항 구체화가 필요한 항목

- 문의 상태 전환 규칙
- 상담자 배정 방식
- 파일 업로드 제약 조건
- 지표 계산 기준
- 다중 스토어 운영 시 데이터 격리 방식

## 4. 후속 문서화 권장 항목

현재 문서 세트 다음 단계로는 아래 문서를 추가 작성하는 것이 적절하다.

- 정보 구조(IA) 문서
- 화면별 와이어프레임 또는 화면 정의서
- ERD 및 데이터 사전
- API 명세서
- 권한 정책 문서
- AI 응답 정책 문서
- 운영자 매뉴얼

## 5. 현 시점 결론

이 프로젝트는 단순한 문의 게시판보다 범위가 넓으며, **자사몰용 통합 고객센터 운영 플랫폼**으로 정의하는 것이 타당하다. 다만 현재 문서는 문제 정의와 요구사항 초안 수준에 가깝기 때문에, 실제 구현에 들어가기 전에는 MVP 범위, 기술 선택, AI 정책, 보안 정책을 우선 확정해야 한다.
