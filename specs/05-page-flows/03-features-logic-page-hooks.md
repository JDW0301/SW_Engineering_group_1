# 페이지별 기능 연결 정리

## 목적

이 문서는 검색, 지표, 대화, 첨부, 요약, 안전 보조, 지식, handoff, 스냅샷 맥락 같은 기능이 각각 어느 페이지에 붙는지 정리한다.

핵심 원칙은 교차 기능을 무조건 새 페이지로 만들지 않고, 사용자의 실제 작업이 일어나는 페이지 안에 자연스럽게 붙이는 것이다.

## 페이지별 기능 매핑

| 기능 | 붙는 페이지 | 설명 |
|---|---|---|
| 문의 검색/필터 | 운영자 문의 목록 페이지 | 목록 triage를 위한 상단 검색/필터 영역 |
| KPI 지표 | 운영자 문의 목록 페이지 | 같은 기간 기준으로 목록과 함께 해석 |
| 문의 대화 타임라인 | 운영자 문의 상세, 고객 문의 상세 | 관리자 응대 대화의 핵심 영역 |
| 챗봇 대화 타임라인 | 고객 챗봇 세션 페이지 | self-service 대화 영역 |
| 메시지 작성 | 운영자 문의 상세, 고객 문의 상세, 고객 챗봇 세션 | 사용자 역할에 따라 다른 입력기 제공 |
| 첨부 미리보기 | 운영자 문의 상세, 고객 문의 상세, 고객 챗봇 세션 | 이미지 썸네일, 확대, 실패 시 다운로드 fallback |
| inquiry summary | 운영자 문의 상세 | 문의 맥락을 빠르게 파악하는 보조 패널 |
| chatbot summary | 운영자 문의 상세 | 연결된 챗봇 맥락 보조 패널 |
| abuse detection | 운영자 문의 상세 | 위험 신호와 표시 완화 정보 참고 |
| FAQ / 정책 / 프리셋 | 운영자 문의 상세, 지식/프리셋 관리 페이지 | 상세에서는 참조, 관리 페이지에서는 유지/업로드 |
| knowledge file upload | 지식/프리셋 관리 페이지 | 챗봇 지식 파일 관리 |
| handoff 요청 | 고객 챗봇 세션 페이지 | 관리자 응대로 연결 |
| linked chatbot context | 운영자 문의 상세 페이지 | handoff 뒤 운영자가 이전 self-service 맥락 확인 |
| customer/order/product snapshot 요약 | 운영자 문의 상세, 고객 문의 상세 | 지원 맥락 보조 정보 |
| manual resync | 운영자 문의 상세, 스냅샷 상세/복구 페이지 | entity 단위 재동기화 |

## 대화와 첨부 기능

### 적용 페이지

- 운영자 문의 상세 페이지
- 고객 문의 상세 페이지
- 고객 챗봇 세션 페이지

### 최소 기대 동작

- 메시지 타임라인 표시
- 발신자 구분
- `content_display` 우선 표시
- 허용 범위 안에서 raw 보기 액션
- 이미지 첨부 썸네일
- 클릭 확대
- 미리보기 실패 시 다운로드 fallback

## 검색과 지표 기능

### 적용 페이지

- 운영자 문의 목록 페이지

### 핵심 원칙

- KPI는 별도 분석 화면보다 문의 목록과 같이 보여야 한다.
- 기간 기준은 단순 preset만 둔다.
- zero-data 상태는 오류가 아니라 데이터 없음으로 표현한다.

## 요약, 안전 보조, 지식 기능

### 적용 페이지

- 주 적용: 운영자 문의 상세 페이지
- 보조 적용: 지식/프리셋 관리 페이지

### 핵심 원칙

- summary와 detection은 최종 판단을 대신하는 기능이 아니다.
- 운영자의 응답 작성과 맥락 파악을 돕는 보조 패널로 붙인다.
- knowledge file 관리만 상대적으로 독립 화면으로 둔다.

## handoff와 챗봇 연결 기능

### 적용 페이지

- 고객 챗봇 세션 페이지
- 운영자 문의 상세 페이지

### 핵심 원칙

- handoff 실행 버튼은 고객 챗봇 세션이 소유한다.
- handoff 이후 연결된 문의는 고객 문의 상세가 후속 대화의 중심이 된다.
- 운영자는 문의 상세에서 연결된 챗봇 맥락을 보조적으로 확인한다.

## 스냅샷과 재동기화 기능

### 적용 페이지

- 운영자 문의 상세 페이지
- 스냅샷 상세/복구 페이지
- 선택적으로 연동 이벤트 로그 페이지

### 핵심 원칙

- snapshot 정보는 보조 정보다.
- 일부 누락 또는 stale 상태여도 문의 처리 자체를 막지 않는다.
- 수동 재동기화는 store 전체가 아니라 entity 단위로 제공한다.

## 구현 우선순위 제안

가장 먼저 구현할 페이지와 기능 조합은 아래 순서가 자연스럽다.

1. 운영자 문의 목록 + 검색/KPI
2. 운영자 문의 상세 + 대화/메모/상태 변경
3. 고객 지원 진입 + 고객 문의 목록/상세
4. 고객 챗봇 세션 + handoff
5. 지식 파일 관리
6. 스냅샷 상세/복구

이 순서는 문의 중심 흐름을 가장 빨리 데모 가능한 상태로 만든다.
