# 운영 지표 및 검색 기준 명세

## 1. 목적

본 문서는 MVP 범위에 포함되는 운영 지표와 검색/필터의 계산 기준, 집계 범위, 해석 규칙을 정의한다.

## 2. 범위

MVP에서 제공하는 운영 지표는 아래 세 가지다.

1. `inquiry_count(문의 수)`
2. `average_first_response_time(평균 응답 시간)`
3. `resolution_rate(해결 완료율)`

고급 SLA 지표, 운영자별 상세 생산성 지표, 복합 BI 지표는 MVP 범위에서 제외한다.

## 3. 공통 집계 원칙

### 3.1 기준 데이터 원천

- 운영 지표는 모두 `inquiry` 기반으로 계산한다.
- `chatbot_session`은 운영 지표의 직접 집계 대상이 아니다.
- 챗봇 self-service에서 즉시 생성된 `inquiry`도 일반 문의와 동일한 집계 대상이 될 수 있다.
- 다만 자동 만료된 문의는 모든 KPI에서 제외한다.

### 3.2 집계 기준 축

- 운영 지표는 **이벤트 발생일 기준**으로 집계한다.
- 각 지표는 자신이 정의한 기준 이벤트 시각을 사용한다.

### 3.3 시간 필드 기준

- `opened_at`: 문의 생성 시각
- `last_message_at`: 가장 최근 메시지 도착 시각
- `last_response_at`: 운영자 측 마지막 응답 시각
- `closed_at`: 종료 처리 시각

지표 계산에 필요한 첫 응답 시각은 `inquiry_message` 이력에서 파생 계산한다.

### 3.4 집계 제외 공통 규칙

아래 문의는 모든 KPI 집계에서 제외한다.

- 자동 만료 처리된 문의
- 테스트/관리 목적의 비운영 문의가 별도 플래그로 식별되는 경우

자동 만료 판단 시각과 테스트 문의 식별 플래그는 **추가 정의 필요**하다.

## 4. KPI 정의

## 4.1 inquiry_count(문의 수)

### 정의

- 집계 기간 내 `opened_at`이 발생한 신규 문의 수

### 기준 이벤트

- 문의 생성 이벤트

### 포함 대상

- 집계 기간 내 새로 생성된 `inquiry`

### 제외 대상

- 재오픈된 기존 문의
- `chatbot_session`
- 자동 만료 처리된 문의

### 해석

- 본 지표는 운영 유입량을 의미한다.
- 재오픈 문의는 운영 부담을 만들 수 있으나, MVP에서는 기본 문의 수에 포함하지 않는다.

## 4.2 average_first_response_time(평균 응답 시간)

### 정의

- 첫 고객 메시지 이후 첫 운영자 응답까지 걸린 시간의 평균

### 기준 이벤트

- 고객 첫 메시지 시각
- 운영자 첫 응답 시각

### 계산 방식

1. 각 문의에서 고객이 남긴 첫 메시지 시각을 찾는다.
2. 해당 문의에서 그 이후 최초로 발생한 운영자 메시지 시각을 찾는다.
3. 두 시각의 차이를 문의별 첫 응답 시간으로 계산한다.
4. 집계 기간 내 기준 이벤트에 해당하는 문의들에 대해 평균을 낸다.

### 포함 대상

- 첫 운영자 응답이 실제로 발생한 문의
- 자동 만료되지 않은 문의

### 제외 대상

- 첫 운영자 응답이 없는 문의
- `chatbot_session`만 존재하고 실제 문의 응답이 없는 경우
- 자동 만료 처리된 문의

### 해석

- 본 지표는 고객이 처음 문의했을 때 얼마나 빨리 첫 응답을 받았는지를 의미한다.
- 후속 응답의 속도는 본 지표에 포함하지 않는다.

## 4.3 resolution_rate(해결 완료율)

### 정의

- 집계 기간 내 종료(`closed_at`)된 문의 수를 같은 기간 내 생성(`opened_at`)된 문의 수로 나눈 비율

### 기준 이벤트

- 분자: 문의 종료 이벤트
- 분모: 문의 생성 이벤트

### 계산식

```text
resolution_rate = (기간 내 종료된 문의 수) / (기간 내 생성된 문의 수)
```

### 분자 규칙

- `closed_at`이 집계 기간 안에 있는 문의를 센다.
- 재오픈 후 다시 종료된 경우에도 **최초 종료 1회만** 종료건수에 포함한다.
- 자동 만료 처리된 문의는 제외한다.

### 분모 규칙

- `opened_at`이 집계 기간 안에 있는 문의를 센다.
- 자동 만료 처리된 문의는 제외한다.

### 해석

- 본 지표는 cohort 해결율이 아니라, MVP 운영 대시보드에서 사용하는 **보수적 처리 성과 지표**다.
- 분자와 분모가 서로 다른 이벤트 축을 사용하므로, 100%를 넘을 가능성은 낮추되 해석 시 주석을 함께 제공한다.

## 5. 재오픈 문의 처리 규칙

- 재오픈 문의는 기본 문의 수에 포함하지 않는다.
- 재오픈 후 첫 응답 시간은 별도 KPI로 계산하지 않는다.
- 재오픈 후 다시 종료된 경우, 해결 완료율 분자에는 최초 종료 1회만 반영한다.

## 6. 검색 및 필터 기준

## 6.1 기본 검색 범위

- 고객명
- 주문번호
- 문의 유형
- 상태
- 기간
- 상품명

## 6.2 기간 필터 규칙

- 시작일은 종료일보다 늦을 수 없다.
- 기간 필터는 지표 집계와 문의 목록 조회에 동일하게 적용한다.
- 기본 기간 프리셋(일/주/월)과 timezone 기준은 **추가 정의 필요**하다.

## 6.3 정렬 규칙

- 문의 목록 기본 정렬은 `최근 메시지 시각 내림차순`이다.
- 미처리 우선 정렬은 `RESOLVED`가 아닌 상태를 상위 배치하는 방식으로 해석한다.

## 7. UI 표시 원칙

- 문의 수는 정수값으로 표시한다.
- 평균 응답 시간은 시간/분 자동 변환 방식으로 표시한다.
- 해결 완료율은 백분율(%)로 표시한다.
- 해결 완료율에는 아래 주석을 함께 둘 수 있다.
  - "같은 기간 내 종료 건수와 생성 건수를 기준으로 계산됨"

초 단위는 표시하지 않는다.

## 8. 예외 처리 원칙

- 집계 대상 문의가 0건이면 문의 수는 `0`으로 표시한다.
- 평균 응답 시간과 해결 완료율은 `-`로 표시한다.
- 외부 데이터 누락은 운영 지표 계산을 직접 막지 않는다. 지표는 `inquiry` 기준으로 계산한다.
- 지표 계산 실패 시 사용자에게는 단순 오류 메시지를 표시하고, 내부 로그에는 실패 원인을 남긴다.

## 9. 추가 정의 필요 사항

- 기본 기간 프리셋(일/주/월)
- timezone 기준
- 테스트 문의 식별 방식
