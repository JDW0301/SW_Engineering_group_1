# 대화 메시지 및 첨부 파일 명세

## 1. 목적

본 문서는 문의 메시지, 챗봇 세션 메시지, 첨부 파일 처리 규칙을 정의한다.

## 2. 대화 모델

## 2.1 inquiry_message

- 사람 상담 문의 맥락에 속한 메시지 단위
- 고객과 운영자의 실제 상담 메시지를 표현한다.

### 필수 필드

- `id`
- `inquiry_id`
- `sender_type`
- `content_raw`
- `content_display`
- `created_at`

## 2.2 chatbot_session

- 고객 self-service 챗봇 대화의 상위 세션 단위
- 문의와는 별도로 저장되며, 필요 시 문의로 전환될 수 있다.

## 2.3 chat_message

- `chatbot_session`에 속한 메시지 단위

### 필수 필드

- `id`
- `chatbot_session_id`
- `sender_type`
- `message_type`
- `content_raw`
- `content_display`
- `sent_at`

## 3. 표시 및 원문 확인 규칙

- 고객 공개 대상 대화는 기본적으로 `content_display`를 표시한다.
- 고객과 운영자는 고객 공개 대상 대화의 원문(`content_raw`)을 확인할 수 있다.
- 내부 메모와 내부 운영 데이터는 원문 공개 대상에 포함되지 않는다.

## 4. 챗봇 세션 생성 및 전환 규칙

- 고객이 self-service 챗봇에 진입하면 `chatbot_session`을 생성한다.
- 상담 전환 버튼 클릭 시 `inquiry`를 즉시 생성한다.
- 운영자는 문의 상세에서 연결된 `chatbot_session`의 전체 원문을 확인할 수 있어야 한다.
- 유의미한 대화가 없고 추가 메시지도 없으면 세션/문의는 10분 후 자동 만료 대상으로 둘 수 있다.

## 5. 첨부 파일 규칙

## 5.1 허용 대상

- 고객 첨부 파일은 이미지 파일만 허용한다.
- 허용 형식은 `jpg`, `jpeg`, `png`다.
- 영상, 문서, 기타 파일은 MVP 범위에서 제외한다.

## 5.2 메타데이터

- `file_name`
- `mime_type`
- `file_size`
- `storage_key`
- `preview_status`
- `uploaded_by`

## 5.3 제약 조건

- 이미지 1개당 최대 크기는 5MB다.
- 고객은 한 메시지당 최대 3장까지 이미지를 업로드할 수 있다.
- 악성 파일 검사 여부는 **추가 정의 필요**하다.

## 5.4 미리보기 규칙

- 첨부 이미지는 메시지 영역에서 썸네일로 표시한다.
- 썸네일을 클릭하면 확대하여 확인할 수 있다.
- 미리보기에 실패하면 원본 파일 다운로드만 제공한다.

## 6. 읽음 여부 및 응답 시간

- `last_response_at`은 운영자 측 마지막 발신 메시지 시각을 기준으로 계산한다.
- 실시간 읽음 여부나 전달 상태를 별도 모델로 세분화할지 여부는 **추가 정의 필요**하다.

## 7. 오류 처리

## 7.1 업로드 실패

- 저장 실패 시 사용자에게 업로드 실패를 명시한다.
- 실패한 첨부는 재업로드 가능해야 한다.

## 7.2 허용되지 않은 파일

- 파일 형식이 허용되지 않으면 업로드를 차단한다.
- 사용자에게 허용 형식 안내가 필요하다.

## 7.3 너무 큰 파일

- 크기 제한 초과 시 업로드를 차단한다.
- 허용 최대 크기를 표시해야 한다.

## 8. 보관 및 삭제

- 첨부 파일 보관 기간은 개인정보 및 보관 정책을 따른다.
- 메시지 삭제 지원 여부는 **추가 정의 필요**하다.
- 삭제 시 물리 삭제와 논리 삭제 중 어떤 방식을 쓸지는 **추가 정의 필요**하다.

## 9. 모듈 책임

- `inquiry_message_service`: 문의 메시지 저장 및 조회
- `chatbot_session_service`: 챗봇 세션 생성, 조회, 전환 처리
- `chat_message_service`: 챗봇 메시지 저장 및 조회
- `attachment_service`: 파일 업로드, 메타데이터 저장, 미리보기 상태 관리

## 10. 추가 정의 필요 사항

- WebSocket 또는 대체 방식 사용 여부
- 첨부 파일 저장소 종류
- 영상 자동 재생/썸네일 정책
