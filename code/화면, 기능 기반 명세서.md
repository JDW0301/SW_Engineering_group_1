### 시작 페이지

1. 로고, 페이지 소개 등을 담는다.
2. 기본적으로 이용자 로그인이 떠있다.
    1. 이용자 로그인 하단의 “회원가입”을 누르면 이용자 회원가입으로 이동
3. 하단에 “관리자라면?”을 누르면 드롭다운으로 관리자 로그인 표현
    1. 관리자 로그인 하단의 “관리자 회원가입을 누르면” 관리자 회원가입으로 이동

### 이용자 회원 가입

1. 개인정보를 작성해 가입
    1. (이상적인 필수 정보들)
    2. 가상 본인인증
        1. 기입된 이름 + 주민번호를 넣고 규칙에 따라 프로젝트용 가상 CI를 생성(형식만)
            1. 주문 DB에서 같은 CI를 가진 주문을 조회할 예정

### 관리자 회원 가입

1. 스토어 정보 기입
    1. 필수 정보 (이상적인 필수 정보들, ex 스토어명, 카테고리(만들어 진것 중 선택), 이름, 전화번호 등)
    2. 추후 입력해도 되는 정보 (이상적인 필수 정보들)
2. 해당 스토어 생성
    1. 페이지, DB 생성

### 이용자 페이지

1. 메인 화면에 “최근 주문 목록”이 n건 표출
    1. 목록 우상단 [주문 목록]을 누르면 주문 목록 페이지 표출
        1. 스토어별, 오래된 순, 최신 순, 기간 범위 별 필터링
    2. 주문 목록을 누르면 해당 스토어 페이지 상담탭으로 이동
        1. 상담 탭에 기본으로 해당 주문이 선택 되어 있음
2. 개인 정보 설정
    1. (이상적인 필수 정보들)
3. 상단에 검색 창
    1. 스토어 탭과 함께 검색창이 열리며 기본적으로 내가 주문 한적 있는 스토어가 먼저 뜸.
        1. 전체, 의류, 음식 등 카테고리별 탭에 스토어 정렬. 클릭 시 해당 스토어 페이지로 이동
    2. 타이핑시 밑에 자동완성

### 스토어 페이지

1. 검색 창은 유지
2. 상단에 스토어 사진, 이름과 함께 간단한 정보 제공
    1. 클릭시 드롭다운으로 설정한 자세한 정보 제공
3. 챗봇 탭, 상담 탭, 문의 탭이 있으며 검색으로 진입시 챗봇 탭이 기본 상태
    1. 챗봇 탭은 챗봇과 채팅할 수 있으며 자주 묻는 질문 프리셋을 제공(스토어가 설정)
        1. 챗봇과 상담 중 답변하지 못하는 내용이 나오면 상담사로 연결하며 정보를 이어서 상담 탭으로 전환
        2. 챗봇과 상담 중 우측 하단의 상담 전환 버튼으로도 전환 가능
    2. 상담 탭은 기본적으로 상단에 상담 가능 시간, 요일 등이 기입되어 있음.(스토어가 설정)
        1. 가능 정보 하단에 + 표시가 있는 긴 칸이 있으며 해당 버튼을 클릭하면 상담하고자 하는 주문 내역 선택 가능
            - 목록ex)일반 상담, 주문 내역1, 주문 내역2, …
        2. 해당 정보를 선택하면 상담사에게 주문 내역 전송하고 상담 채팅 시작
    3. 문의 탭은 게시판의 형태로 되어있으며 문의 작성 버튼을 통해 작성할 수 있다.
        1. 문의 작성 시 제목(없으면 본문 내용으로), 본문, 주문 내용 참조, 이미지 첨부(미리보기 제공), 비밀글 체크 박스, 글 작성
        2. 목록 카드는 문의 번호, 날짜, 제목(비밀 글 일시 “비밀글 입니다.”), 상태(완료, 대기), ID(일부 마스킹)을 표현
        3. 게시글은 제목, 날짜(+ 시간), 참조된 주문, 작성자(일부 마스킹), 본문, 제목, 답변, 수정 버튼, 목차 버튼으로 구성
        4. 관리자가 답변 작성시 상태 완료로 변경
    4. 나의 문의 탭은 버튼을 통해 나의 상담과 문의를 기록을 스위칭해서 확인할 수 있음.
        1. 상담을 누르면 해당 채팅창으로 이동(종료된 상담은 내역 확인만 가능)
        2.  게시글을 누르면 해당 게시글로 이동

### 관리자 페이지

1. 메인화면
    - 페이지 이름은 “{스토어명}(큰 글씨) : 관리자(작은 글씨)”
    1. 통계 섹션
        1. 상담 완료율이 원형 진행도로 표시되는데 가운데 흰 원 사이 완료 수/ 총 수를 표시
            1. 하단에 있는 7일(디폴트), 30일, 전체 를 선택해 완료율 표현
        2. 문의량을 숫자로 표시
            1. 하단에 있는 1일(디폴트), 7일, 30일, 전체를 선택해 표현
    2. 최근 상담 섹션은 최근 상담 카드(고객이 상담을 열면 생기기는 카드)를 5개 까지 표현
        1. 최신 메세지 순으로 정렬
        2. 클릭 시 채팅창이 열림 
    3. 최근 문의 섹션은 최근 순으로 문의 5가지를 표현 카드 우측 상태(완료, 대기) 표기
        1. 클릭시 게시글 표현 및 답변 작성 가능
    4. 주문 목록 섹션
        1. 최근 순으로 주문 목록 10개 표현
            1. 우측 < 모양의 토글 클릭시 해당 주문과 연결되어있는 상담, 문의 탭이 달린 리스트 표현
2. 설정을 들어가면 챗봇 관리, 스토어 관리가 있음.
    1. 챗봇 관리는 챗봇 적용 정보 관리, 프리셋 관리로 나뉨
        1. 챗봇 적용 정보 관리는 txt 파일을 업로드 할 수 있으며 업로드한 txt는  제목,  크기 리스트 형태로 정리됨.
            1. 클릭하면  txt 프리뷰
            2. 리스트 항목의 끝단에 있는 X를 누르면 확인/취소 버튼이 뜨고 확인을 누르면 삭제
            3. 드래그 드롭, 형태로 txt 파일을 넣을 수 있음.
        2. 프리셋 관리는 자주 묻는 질문을 입력하고 답변을 미리 등록해 둘 수 있음.
            1. 저장을 누르면 반영.
            2. 최대 5개 가능하며 초기화 버튼을 클릭 시 해당 프리셋 리셋.
            3. 해당 자주 묻는 질문은 챗봇과 대화시 나열되며 클릭하면 미리 등록된 답변을 제공.
    2. 스토어 관리는 전화번호, 주소, 상담 운영 시간, 소개글 등을 정해진 입력란에 작성할 수 있음.
        1. 저장 전 프리셋 버튼을 통해 적용 모습 확인 가능
            1. 챗봇, 상담, 글 작성 등 페이지 외관 제외한 것들의 적용 시 화면을 확인할 수 있음.
        2. 저장 클릭 시 DB에 반영 
3. 상담·문의 관리 채널
    1. 스토어 상담 채팅 내역(탭)
        1. 상담 리스트 카드 표현
        2. 최신순(디폴트), 오래된 순, 기간(범위), 이름 검색, ID 검색 필터링
    2. 스토어 문의 내역(탭)
        1. 문의 리스트 표현
        2. 최신순(디폴트), 오래된 순, 기간(범위), 이름 검색, ID 검색 필터링
4. 통계 채널
    1. 상담 완료율이 원형 진행도로 표시되는데 가운데 흰 원 사이 완료 수/ 총 수를 표시
        1. 하단에 있는 7일(디폴트), 30일, 전체 를 선택해 완료율 표현
    2. 문의 완료류
    3. 문의량을 숫자로 표시
        1. 하단에 있는 1일(디폴트), 7일, 30일, 전체를 선택해 표현
    4. 상담 평균 응답시간(첫 메세지 이후 답장까지)
    5.  챗봇 → 상담 전환율
5. 주문 목록 섹션
    1. 문의 리스트
        1. 상담 리스트 카드 표현
        2. 최신순(디폴트), 오래된 순, 기간(범위), 이름 검색, ID 검색 필터링

### 채팅[챗봇, 상담](이용자/관리자)

1. 이용자
    1. 챗봇 
        1. 메세지 입력, 이미지 삽입이 가능하다.
        2. 챗봇에서는 질문 프리셋이 제공되며 클릭시 해당 질문을 하고 매핑된 채팅이 제공된다.
        3. 챗봇이 대답할 수 없는 정보를 요청시 상담 전환을 요청한다.
            1. 상담 전환 버튼이 존재
        4. 상담 전환을 누르면 해당 대화 세션 그대로 상담으로 넘어간다.
    2. 상담 
        1. 상담은 “완료”, “종료”, “진행 중” 3가지 중 하나의 세션을 가지는데 이용자가 생성하면 진행 중 상태로 생성된다.
        2. 이용자가 아무 입력을 하지 않거나 상담자가 마지막으로 입력했고 3시간 동안 응답이 없을시  “종료”로 바뀌며 입력할 수 없는 상태로 전환
            1. 전환 시 “상담자의 답변에 3시간 동안 응답이 없을시 상담이 종료됩니다.” 표현
2. 관리자
    1. 메세지 입력, 이미지 삽입이 가능하다.
    2. 완료 버튼을 누름으로서 상담을 “완료”상태로 만들 수 있으며 관리자 또한 채팅을 입력할 수 없는 상태로 바뀜.
